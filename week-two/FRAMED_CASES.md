## Case Study 1 — Academic Planner App

### Problem

For the Algorithm Design I course, I developed an application to help students organize assignments, exams, and academic deadlines in a single place. Beyond simply completing the course project, I set a personal challenge for myself: to create a full-fledged React application that balanced usability, organization, and maintainable code.
### What I Did

Designed and developed the application using React, creating reusable components and organizing the interface around the daily workflow of a student rather than around technical features.

Made decisions about navigation, task organization, and responsive layouts to keep the application easy to use across different screen sizes.

Used AI throughout development to brainstorm implementation approaches, review code, troubleshoot issues, and improve documentation while making the final engineering decisions myself.

### Outcome

The project demonstrates my ability to take an idea from planning through implementation, balancing technical requirements with user experience. It also reflects how I use AI to accelerate development without replacing my own problem-solving process.

## Case Study 2 — Sentiment Analysis Hackathon

### Problem

Our team needed to build a sentiment analysis application that allowed users to submit text for machine learning predictions while providing a reliable backend that could communicate with the frontend and the prediction service.

### What I Did

Worked as part of the backend team using Java and Spring Boot, contributing to the development of REST APIs, request validation, data transfer objects, and application structure.

Collaborated with teammates to integrate backend services with the frontend and support the overall application workflow. Throughout development, I used AI to explore implementation options, investigate errors, and review code while validating every solution before applying it.

### Outcome

The project strengthened my experience working in a collaborative development environment and demonstrated my ability to contribute to AI-related software beyond the user interface. It also reinforced the importance of communication, API design, and iterative problem-solving within a team.

## Case Study 4 — AI-Assisted Portfolio Planning (FlyRank AI Internship)

### Problem

Many developer portfolios list technologies and projects but do not provide enough evidence for a hiring manager to understand how the developer thinks, makes decisions, or uses AI responsibly during software development.

### What I Did

Defined a clear proof statement, identified a specific audience, established a single call to action, and designed a sitemap where every page contributes to the same objective.

Used AI as a collaborative thinking partner to challenge assumptions, review the structure, improve information architecture, and strengthen the narrative connecting each project. Rather than accepting generated content, I continuously refined the material to reflect my own experience and engineering decisions.

### Outcome

The planning process produced a clear strategy for a portfolio built around evidence instead of claims. It demonstrates how I approach AI as a tool for critical thinking and iterative improvement rather than as a replacement for my own judgment.
