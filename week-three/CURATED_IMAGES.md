# Real Images (Preferred)

I intentionally chose **real images** whenever authenticity is important.

These include:

* Professional portrait
* Portfolio screenshots
* Application screenshots
* Project interfaces
* UI demonstrations
* Any image intended as evidence of completed work

Real captures are more trustworthy than AI-generated stand-ins and better support the portfolio's purpose of demonstrating real engineering work.

---

# Rejected Image

I initially considered using an AI-generated banner that included decorative programming illustrations, interface mockups, and branding elements similar to my previous LinkedIn cover.

I rejected it because it contained too many focal points and competed with the content instead of supporting it. It also felt more illustrative than editorial, making the page appear busier than intended. I replaced it with a simpler abstract background that reinforces the portfolio's identity without distracting from the projects.

---

# Image Style Guidelines

All generated imagery follows a single visual direction:

* Editorial
* Modern
* Minimal
* Dark neutral backgrounds
* Rose accent color
* Blue technical accent
* Soft lighting
* Clean geometric compositions
* Subtle UI-inspired elements

This consistency ensures that AI-generated visuals feel like part of one design system rather than unrelated assets.

---

# Reflection

This exercise reinforced that AI is most valuable for creating supporting visuals, not replacing evidence of real work. Throughout the portfolio, I prioritized authenticity by using real photographs for personal representation and real screenshots for projects, while limiting AI-generated images to decorative elements that enhance the overall visual identity.
