# Frontend AI Engineering – Homework Submission

## Project

**AI Task Manager**

A task management application developed with **React** and **Vite**, using **GitHub Copilot** as an AI development assistant throughout the project lifecycle.

---

## 1. Prompts Used

### Prompt 1 – Project Architecture

```text
You are a senior Frontend Engineer and React mentor.

I am completing an AI Engineering internship assignment whose goal is to demonstrate how AI can be used as a development assistant rather than simply generating code.

I want to build a Task Management application using React.

Before writing any code, help me design the project.

Requirements:
- Use React with Vite.
- Use modern JavaScript.
- Keep the architecture simple but scalable.
- Follow React best practices.
- Use functional components and Hooks.
- Organize the project with reusable components.
- Keep the code easy to understand for junior frontend developers.
- Avoid unnecessary libraries.
- Explain every important architectural decision.

The application should include:
- Add a new task
- Edit a task
- Delete a task
- Mark tasks as completed
- Filter tasks
- Search tasks
- Persist tasks using Local Storage
- Responsive layout

Do not generate code yet.

Instead provide:
- Folder structure
- Components
- State management
- Data model
- Implementation plan
- Future improvements
- Architecture explanation
```

## 2. How AI Helped Me

GitHub Copilot was used as a development assistant throughout the project rather than as a tool to generate the entire application automatically.

AI contributed by:

* Designing the initial application architecture.
* Suggesting a scalable folder structure.
* Recommending reusable React components.
* Helping implement custom hooks.
* Assisting with CRUD operations.
* Suggesting improvements for accessibility.
* Performing a code review.
* Recommending automated tests.
* Explaining architectural decisions and React best practices.

Using AI iteratively made it easier to organize the project while still requiring me to understand and validate every implementation.

---

## 3. Manual Improvements and Corrections

Throughout the project, I reviewed the code generated by GitHub Copilot instead of accepting it without verification.

My manual work mainly involved:

* Validating the generated code before integrating it into the project.
* Refining prompts to improve the quality of the generated solutions.
* Identifying and fixing a missing stylesheet import (`Header.css`) that prevented the header styles from being applied correctly.
* Improving the task statistics section by redesigning it into card-style elements with better spacing and visual hierarchy.
* Making small adjustments to the user interface and application layout.
* Testing the application and correcting minor issues when necessary.
* Reviewing the AI suggestions and deciding which recommendations should be applied.

Rather than rewriting large portions of the code, my role was to guide the AI, validate its output, and ensure the final implementation satisfied the project requirements.

---

## 4. Lessons Learned

This project demonstrated that AI is most effective when used as a collaborative development assistant rather than as a replacement for software engineering knowledge.

Key takeaways:

* Well-structured prompts produce significantly better results.
* Architectural planning before coding reduces future refactoring.
* AI-generated code should always be reviewed and validated.
* Manual improvements remain essential for maintainability.
* AI increases productivity while developers remain responsible for design decisions and code quality.
